## 3. Опис реалізованих алгоритмів

### 1. Insertion Sort
* **Як виконується:** Алгоритм віртуально ділить масив на відсортовану (ліву) та невідсортовану (праву) частини. Він послідовно бере кожен елемент з невідсортованої частини і шукає для нього правильне місце у відсортованій, "вставляючи" його туди.
* **Ключові етапи:**
    1. Вибір поточного елемента (ключа).
    2. Порівняння ключа з елементами ліворуч від нього.
    3. Зсув усіх більших (або менших, залежно від компаратора) елементів на одну позицію вправо.
    4. Вставка ключа на звільнене місце.
* **Алгоритмічна складність:**
    * **Найкращий випадок:** $O(N)$ (якщо масив вже відсортований, відбувається лише одне порівняння на кожен елемент без зсувів).
    * **Найгірший випадок:** $O(N^2)$ (якщо масив відсортований у зворотному порядку).

### 2. Quicksort
* **Як виконується:** Працює за принципом "Divide & Conquer". Вибирає опорний елемент, розбиває масив так, щоб усі елементи з одного боку були менші за півот, а з іншого — більші, після чого рекурсивно або через стек застосовує ту саму логіку до обох частин.
* **Ключові етапи:**
    1. **Вибір півота (Median-of-three):** Береться перший, середній та останній елементи підмасиву, і опорним стає той, що є медіаною. Це запобігає деградації алгоритму на вже відсортованих масивах.
    2. **Hoare Partition:** Два вказівники рухаються назустріч один одному з кінців підмасиву. Якщо лівий знаходить елемент, більший за півот, а правий — менший, вони міняються місцями.
    3. **Обробка підмасивів:** Виклик алгоритму для лівої та правої частини (у проєкті реалізовано як рекурсивну версію, так і оптимізовану через явний `std::stack`).
* **Алгоритмічна складність:**
    * **Найкращий/Середній випадок:** $O(N \log N)$.
    * **Найгірший випадок:** $O(N^2)$ (вкрай рідко завдяки підходу median-of-three).

### 3. Merge Sort
* **Як виконується:** Також використовує стратегію "Divide & Conquer". Масив рекурсивно ділиться навпіл, поки не залишаться списки з 1 елемента (які вважаються відсортованими). Потім ці малі списки "зливаються" у більші відсортовані списки.
* **Ключові етапи:**
    1. Знаходження середини масиву та його поділ на дві підгрупи.
    2. Рекурсивний спуск до базового випадку (розмір 1).
    3. **Злиття:** Створення тимчасових масивів (або використання буфера), поелементне порівняння двох підмасивів і перенесення меншого елемента в оригінальний масив.
* **Алгоритмічна складність:**
    * **Найкращий випадок:** $O(N \log N)$.
    * **Найгірший випадок:** $O(N \log N)$. (Додаткова пам'ять: $O(N)$).

### 4. Combined Sort
* **Як виконується:** Це гібридний алгоритм, який використовує переваги обох підходів. Для великих масивів використовується ефективний Quicksort, але коли розмір підмасиву під час поділу стає дуже малим, накладні витрати на виклики функцій (чи стеку) стають невиправданими. Тоді алгоритм перемикається на Insertion Sort.
* **Ключові етапи:**
    1. Перевірка розміру поточного підмасиву (чи він менший за вказаний `threshold`, наприклад, 16).
    2. Якщо так — виклик Insertion Sort для цього підмасиву.
    3. Якщо ні — виконання Hoare Partition і подальше розбиття.
* **Алгоритмічна складність:**
    * **Найкращий/Середній випадок:** $O(N \log N)$.
    * Працює швидше за чистий Quicksort на 10-20% через оптимізацію роботи з малим кешем процесора.

### 5. In-Place File Sorting
* **Як виконується:** Виконує сортування бінарних структур фіксованого розміру (конкретно у моєму прикладі - пакетів) безпосередньо на жорсткому диску, не використовуючи оперативну пам'ять для зберігання масиву. Базується на механіці Insertion Sort з використанням зміщень.
* **Ключові етапи:**
    1. Читання розміру структури з кастомного бінарного заголовка.
    2. Використання `seekg` (читання) та `seekp` (запис) для переміщення вказівника диска замість використання індексів оперативної пам'яті.
    3. Завантаження лише поточного та порівнюваного елементів у мікро-буфери (`vector<char>`) в пам'ять.
* **Алгоритмічна складність:**
    * Операції I/O: $O(N^2)$ (дискові операції найповільніші).
    * Споживання пам'яті: $O(1)$ (константне, дозволяє сортувати терабайти даних).

## 4. Приклади використання алгоритмів у реальному світі

**1. Insertion Sort**
* **Real-time системи (Сортування "на льоту"):** Коли дані надходять безперервно один за одним (наприклад, читання логів), Insertion Sort ідеально підходить для підтримки відсортованого стану, оскільки додавання одного нового елемента у вже відсортований масив займає всього $O(N)$ часу.
* **Складова гібридних алгоритмів:** У стандартних бібліотеках мов програмування Insertion Sort завжди використовується як фінальний етап для сортування маленьких шматків даних (до 10-32 елементів), оскільки він не витрачає час на рекурсію або додаткову пам'ять.

**2. Quicksort**
* **Системні бібліотеки (Загальне призначення):** Класичний вибір для імплементації вбудованих функцій `sort()` у багатьох мовах. Завдяки тому, що алгоритм сортує дані in-place (без великого об'єму додаткової пам'яті), він максимально ефективно використовує L1/L2 кеш сучасних процесорів.
* **Ігрові рушії та 3D графіка:** Використовується для сортування об'єктів за відстанню до камери (Z-ordering) перед рендерингом.

**3. Merge Sort**
* **Сортування Linked List'ів:** На відміну від масивів, зв'язні списки не мають швидкого випадкового доступу за індексом (`list[5]`). Merge Sort оперує лише послідовними вказівниками, що робить його найшвидшим алгоритмом ($O(N \log N)$) для сортування таких структур.
* **Зовнішнє сортування баз даних (External Sorting):** Якщо потрібно відсортувати таблицю бази даних на 500 ГБ, а на сервері лише 16 ГБ оперативної пам'яті, СКБД (наприклад, PostgreSQL) ділить таблицю на шматки, сортує їх у пам'яті, записує в тимчасові файли, а потім застосовує потокове злиття (Merge) цих файлів у єдиний фінальний файл.

**4. Combined Sort**
* **Обробка великих даних:** У системах аналітики, де потрібно сортувати мільйони записів і де кожен відсоток продуктивності заощаджує гроші на хмарних серверах, гібридні алгоритми є обов'язковим стандартом, оскільки вони усувають вузькі місця чистих алгоритмів.
* **V8:** V8 тривалий час використовував гібридний підхід (Quicksort + Insertion Sort для коротких масивів) для методу `Array.prototype.sort()`.
